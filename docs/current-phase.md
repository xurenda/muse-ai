## 如何更新进度（v0.2 及以后）

本文件只做**快照**（当前版本、链接指向哪）。大版本交付后收敛为 `docs/releases/vX.Y.md`；进行中的任务可新建 `docs/v0.2/` 等目录追踪，**不必**长期保留 phase 文档在主索引。

### 开始一个新版本 / 阶段

1. 更新本文顶部：**最后更新**、**目标版本**、**当前**
2. 在 `docs/v0.N/`（可选）维护进行中的任务清单
3. **下一步** 链到当前工作文档或 roadmap 对应节

### 完成一个大版本

1. 撰写 `docs/releases/vX.Y.md`（已交付、未做、设计决策摘要）
2. 将进行中文档移至 `docs/archive/`（若有）
3. 更新 [roadmap.md](./roadmap.md)、[docs/README.md](./README.md)
4. 更新本文：**当前** → 下一版本规划
