# v0.1.1 阶段文档

v0.1 MVP 交付后的**第一个增量版本**：补齐模型策略（分组 fallback + 任务路由 + 设置页），提升自用可靠性与成本可控性。

> **协作说明**：进行本版本时，在此期间有什么问题，我们都可以进行讨论。当前进度见 [current-phase.md](../current-phase.md)（v0.1 收尾完成后切换指向本目录）。

| 阶段 | 文档                       | 名称                    | 状态      |
| ---- | -------------------------- | ----------------------- | --------- |
| 1    | [phase-1.md](./phase-1.md) | 模型策略与设置          | ✅ 已完成 |
| 2    | [phase-2.md](./phase-2.md) | Server 侧模型解析与代理 | ✅ 已完成 |

**v0.1.1 总目标**：用户可配置高/中/低模型池（组内有序 fallback）；聊天可选 tier 或具体模型；标题生成、压缩等后台任务可独立指定模型；设置页与聊天选择器与之对齐。**阶段 2** 已将解析与 fallback 下沉至 Server LLM 代理，CLI 不再拉取 `model-strategy`。

**前置**： [v0.1 MVP](../v0.1/README.md) 阶段 7 收尾（含连续自用验收）。

**明确不做（本版本）**：

- 聊天 **Auto** 模式（按问题难度自动选 tier）
- Auto 路由专用模型配置
- Backend 持久化 token 统计、市场上传、通用 MCP

总览与更远期规划见 [路线规划](../roadmap.md)。
