## 一句话

一个人、一台（或多台）电脑、一个 Web 页，能组装 agent、直连 CLI 聊天、看 Session 树；API Key 在后端；支持模型池 fallback、长对话 compact、断线恢复与用量观测。

---

## 已交付能力

### 聊天与 Agent

- Web **SSE 直连 CLI** 流式对话（不经 Backend 转发）
- Markdown 渲染、thinking / tool call 可视化、Shiki 代码高亮
- **steer**（Enter）与 **follow_up**（Shift+Enter）真接入；idle 时 Enter 为正常 prompt
- 会话级 Agent、model、tier、thinking level 切换
- Persona + Skills 组装 Agent；内置 Persona ×2–3、Skill ×2–3

### Session

- 树形 Session（pi AgentHarness JSONL），分叉、navigate、Fork
- 持久化在 CLI 本地 `~/.muse/sessions/`；重启 CLI 后会话仍在
- **手动 compact** + overflow **自动 compact**；压缩中 UI 禁用 + toast
- 断线后 **resync**（`getSessionTree` + 收尾 stale streaming/tool 尾部）

### 设备与连接

- Backend 设备配对、心跳、endpoint 注册
- AppLayout **底栏**聚合状态：就绪 / 不可达 / 会话断开 / 重连中
- CLI **设备级 SSE**（`/device/events`）：可达性、Session 列表刷新、优雅关机通知
- 不可达时 **禁发**；错误 i18n；面板内重试与自动重试
- 无设备时 `/chat` 步骤化引导

### 模型策略（v0.1.1）

- 高 / 中 / 低 **模型池**，组内有序 **fallback**（429、503、超时等可恢复错误）
- 聊天可选 **tier** 或 **具体 modelRef**；压缩、标题生成等任务独立路由或「跟随对话」
- **`/settings/models`**：模型池编辑 + 任务路由表
- **Server LLM 代理**负责 expand pools + fallback；CLI 注入 `X-Muse-*` 头；Web 以 SSE `model_resolved` 为准

### 用量与上下文（v0.1.1）

- **Session 累计 token**（↑/↓/total，可选 cost / cache）
- **上下文窗口占用**（`ContextUsage`）：底栏紧凑摘要 + 点击展开面板
- 输入区只保留 Agent 选择与撰写；压缩按钮移至上下文面板
- 单条消息 token / 耗时统计（assistant 消息区展示）

### Backend 控制面

- 用户注册 / 登录（JWT）
- LLM Provider CRUD（加密存 Key）；CLI 无 Key 经代理调模型
- 设备配对码、在线列表、endpoint 目录
- 模型策略配置 API（`user_settings`）

### CLI Runtime

- `muse start` HTTP + SSE daemon；`muse pair` / `muse agent` 等
- `MuseHarness` 封装 AgentHarness（`packages/core` 唯一引用 pi）
- 内置 Tools：read / ls / bash / write / edit / grep / find（grep/find 依赖系统 PATH）
- turn-scoped Harness：每轮 prompt 借还；steer/follow_up 仅 turn 进行中

### Web UI

- light / dark / system 主题；可伸缩三栏布局
- `/settings/*`：Providers、Devices、Agents、Models、主题
- 新对话页居中输入、首条消息建会话；聊天区智能滚动

- [architecture.md](../architecture.md)、[protocols.md](../protocols.md)、[product.md](../product.md)

## 重要设计决策

| 项               | 决策                       | 理由                                             |
| ---------------- | -------------------------- | ------------------------------------------------ |
| 聊天流           | Web → **CLI 直连** SSE     | 低延迟；Backend 不转发聊天                       |
| LLM Key          | 仅存 **Backend**           | CLI 只有 user/device 凭证                        |
| pi 依赖          | 仅 **`packages/core`**     | 升级隔离                                         |
| Harness 生命周期 | **turn 结束即释放**        | HTTP daemon 对齐 steer 语义；上下文在 pi Session |
| 设备状态         | **设备 SSE** + 底栏聚合    | 替代 health 轮询；推送 Session 目录变更          |
| 模型解析         | **Server 代理层** fallback | v0.1.1；CLI 不拉 model-strategy                  |
| Session 存储     | CLI 本地 JSONL             | v0.1 不上传 server                               |
| MCP              | `~/.muse/mcps/` **占位**   | 第一期不做通用 MCP 框架                          |
| Auto 选模型      | **不做**                   | 类型可预留，UI 与路由均不启用                    |

更多协议与存储细节见 [architecture.md](../architecture.md)。
